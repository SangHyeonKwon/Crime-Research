## 개요

ChainOpera AI 는 Google Cloud · Microsoft Azure · Samsung 협업을 내세운 AI×블록체인 풀스택 프로젝트로, 2025년 9월 25일 4개 CEX (Binance Alpha · Bybit · Bitget · Gate) 와 1개 DEX (Aster) 에 동시 상장되며 한때 FDV $18B 대에 도달했다. 다른 케이스들과 결이 다른 지점은 증거의 성격이다. RAVE · SIREN · M 이 지갑 흐름과 가격 데이터를 근거로 한다면, COAI 에서 ZachXBT 가 지목한 것은 **컨트랙트 코드 자체** — proxy contract 의 admin 권한이 아직 포기되지 않아, 팀이 토큰 동작 로직을 *언제든* 바꿀 수 있는 구조다.

이 사실은 추정이 아니다. BscScan 에서 누구나 검증할 수 있는 공개 정보다. 그래서 본 페이지는 *과거의 단일 사건*을 복기하지 않는다. 상장 직후 고점에서 수개월에 걸친 하락은 있었지만, RAVE 처럼 24시간 안에 -95% 가 찍힌 단발성 청산 캐스케이드도, 거래소 조사 발표도 아직 없다. 핵심은 가격 곡선이 아니라 **여전히 살아 있는 컨트랙트 권한** — manipulation 에 취약한 구조적 조건이 *지금* 활성화되어 있다는 점이다.

## 수법 분석 — "기술적 시그널" 패턴

엔지니어가 아닌 트레이더에게 proxy contract 는 익숙하지 않은 개념이라 먼저 짚어둔다. 일반 컨트랙트는 로직과 저장소가 한 곳에 있고 한 번 배포되면 바꿀 수 없다. proxy 패턴은 이를 둘로 쪼갠다 — 주소가 고정된 Proxy(저장소) 가 사용자의 호출을 받아, 교체 가능한 Implementation(로직) 에 위임한다.

proxy 자체는 정당한 기술이다. 보안 취약점이 발견됐을 때 토큰을 재발행하지 않고 고칠 수 있고, DeFi 프로토콜의 점진적 개선을 가능하게 한다. 문제는 그 교체 권한, 즉 admin 이 *살아 있는 상태*일 때 생긴다.

정상적인 프로젝트는 출시 후 일정 기간이 지나면 admin 권한을 zero address 로 넘기거나(renounce), multisig + timelock 으로 묶어 변경 시 24~48시간 전 공지가 강제되도록 한다. COAI 는 ZachXBT 폭로 시점 기준 이 중 어느 것도 명시적으로 적용하지 않은 상태였다. 이는 "팀이 권한을 가지고 있다" 를 넘어, **팀이 마음먹으면 사용자 자산을 0으로 만들 수 있는 경로가 닫혀 있지 않다**는 뜻이다. 본 페이지는 ChainOpera 가 그 권한을 *행사했다*거나 *행사할 것이다*라고 단정하지 않는다. 다만 그 경로가 검증 가능한 형태로 존재한다는 사실을 짚는다.

### 자체 측정 — 권한 이벤트 전수 추적

ZachXBT 의 "ownership 미포기" 주장을 재인용에 그치지 않고, proxy(`0x0A8D…6EA5`) 와 implementation(`0xa9d0…a877`) 의 EIP-1967 권한 이벤트(`Upgraded` · `AdminChanged` · `OwnershipTransferred`)를 배포 시점부터 현재까지 전수 인덱싱했다 (`bnb.logs`). 전체 이력은 **단 2건, 둘 다 genesis 블록**(2025-09-22 09:52:10 UTC)에서만 발생했다:

- `Upgraded` → implementation `0xA9D0…A877` 설정 — 본문 명시값과 일치 (검증됨)
- `OwnershipTransferred` → `0x0000…0000` 에서 `0xAC17F51B2F5BDBC63132AB434F92C1A36D2DF10C` 로 이전

그 이후 마지막 인덱싱 블록(2026-05-16)까지 **추가 `Upgraded` 0건, `AdminChanged` 0건, zero address 로의 `OwnershipTransferred` 0건**이다. 정리하면:

| ZachXBT 주장 | 자체 Dune 측정 |
|---|---|
| proxy ownership 미포기 | 소유권이 genesis 에 `0xAC17…F10C` 로 설정된 뒤 **zero address 이전 이벤트 0건** → 미포기 독립 확인 |
| (자체 추가) implementation 교체 가능성 | 교체 권한은 살아 있으나 **genesis 1회 설정 후 실제 교체 0건** — 잠재적 위험이지 *실현된* 위험은 아님 |
| (자체 추가) admin 통제 | `AdminChanged` 0건 — admin slot 배포 후 무변경 |

방법론 한계는 정직하게 적어둔다. Dune 은 임의 storage slot 을 직접 못 읽어 *현재* owner 는 마지막 이벤트로 역산한 값이다. 또 `0xAC17…F10C` 가 단일 키(EOA)인지 multisig·timelock 인지는 이벤트 로그만으로 판정되지 않는다 — 위험의 *경중*은 이 주소의 성격에 달려 있고, 그 확인은 트레이더의 몫으로 남는다.

### 두 번째 시그널 — 시총 vs 거래량

기술적 시그널 외에 시장 행동도 비정상 범위에 있다. ZachXBT 는 COAI 가 시총 $315M 에 일거래량 $5M, 비율 약 63:1 이라 "demand 가 거의 없음을 시사" 한다고 표현했다. 정상 mid-cap 은 20~50:1 범위이고, 비교 삼아 BTC 가 약 25:1, ETH 가 약 30:1 이다. 거래량이 이 정도로 얇다는 것은 자연스러운 시장 참여가 부족하고, 시총이 *거래되지 않은 채 유지되고* 있다는 신호로 읽을 여지가 있다.

### 인상적인 외양 — 펀더멘털 narrative

COAI 의 펀더멘털 서사는 정교하다. Google Cloud · Microsoft Azure · Amazon 클라우드 인프라, Samsung · Toyota 엔터프라이즈 협업, EigenLayer · Babylon 크립토 인프라 협업을 내세우고, 출시 시점에 AI Terminal 일일 활성 100만 · 유료 15만 · 커뮤니티 AI 에이전트 1,000개+ 같은 지표를 발표했다.

이런 수치가 *진짜*인지 확인할 방법은 제한적이다. Google · Microsoft · Samsung 측의 *공식 발표*는 대부분 확인되지 않고 ChainOpera 측 발표만 존재한다. 100만 DAU 에는 incentivized 캠페인 사용자가 포함됐을 수 있고, AI 에이전트 1,000개가 *실제 사용*인지 *제출만*인지도 외부에서 가리기 어렵다. 검증 부담이 트레이더에게 넘어와 있고, 자체 발표 수치만으로 평가하기는 어렵다는 점이 핵심이다.

### 컨트랙트 직접 검증

이 케이스의 강점은 독자가 직접 확인할 수 있다는 데 있다.

1. BscScan 에서 `0x0A8D6C86e1bcE73fE4D0bD531e1a567306836EA5` 검색 → "EIP-1967 Transparent Proxy pattern" 표시 확인
2. Implementation `0xa9d0b1770ad65cbc4f5dffc0f24f42c57933a877` 의 로직 verified 여부 확인
3. "Read as Proxy" 탭에서 admin 주소 확인 — zero address(`0x0000...0000`) 면 안전, EOA 나 multisig 면 해당 주체가 통제 가능
4. admin 이 multisig 라면 서명 수(1-of-N? 5-of-N?)와 timelock 적용 여부 확인

## 교훈 — 다음 비슷한 케이스를 식별하는 시그널

COAI 는 다른 케이스들과 달리 **온체인에서 검증 가능한 정량적·구조적 시그널**을 제공한다. 지갑 흐름 추정이 아니라 코드로 확인되는 사실이라 식별 난이도가 낮다.

### 기술적 시그널 (가장 명확)

- proxy 패턴 사용 자체는 정상 — 단, admin 검증이 반드시 동반돼야 함
- admin 이 EOA(개인 지갑) 면 한 명의 키 보유자가 전권 — 최고 위험
- admin 이 multisig 라도 timelock 이 없으면 즉시 변경 가능 — 중간 위험
- admin 이 zero address 로 renounce 됐으면 변경 불가 — 안전
- Implementation 이 verified 되지 않았으면 로직 자체가 블랙박스 — 매우 위험
- `mint` · `blacklist` 함수는 미사용이라도 *존재 자체*가 risk

### 펀더멘털 검증 시그널

- 거대 기업 협업 발표는 *상대방 공식 발표* 존재 여부로 교차 확인
- DAU/MAU 지표는 incentivized 캠페인 사용자와 organic 사용자를 구분
- GitHub 활동으로 코드베이스가 실제 active development 중인지 확인

### 시장 행동 시그널

- 시총 / 일거래량 비율 50:1 이상이면 자연스러운 demand 부족 의심
- 상장 당일 perp 거래량이 spot 의 100배+ 면 derivatives 주도 거래
- 단일일 perp 거래량이 시총보다 크면 비정상적 turnover

### 상장 패턴 시그널

- 3개+ 거래소 동시 상장은 cross-exchange coordinated 셋업 가능성
- Binance Alpha + spot/futures 동시 상장은 alpha-no-spot 변형
- 에어드롭이 Alpha Points 같은 인사이더 친화적 채널로 분배되면 분배 신원 세탁 여지

### 매수 전 5분 체크리스트

COAI 에서 배운 절차는 모든 토큰에 적용된다. 컨트랙트 주소 검색 → verified 여부 → proxy 인가(맞으면 implementation 확인) → owner/admin 주소 → zero address 인가 → mint/blacklist/tax 함수 존재 → 유동성 풀 lock → 상위 holder 분포. 이 중 하나라도 빨간 깃발이면 매수 전에 다시 생각한다. 시총 $315M 도, top 거래소 상장도 안전을 보장하지 않는다.

### "잠재적" 위험과 "실현된" 위험

COAI 가 주는 가장 중요한 교훈은 *manipulation 이 일어나기 전*에도 인프라가 위험할 수 있다는 점이다. "아무 일도 안 일어났으니 안전하다" 는 잘못된 판단이다. 컨트랙트 audit 이 단순 코드 버그 검사가 아니라 *권한 구조* 검사라는 의미이기도 하다.
